## Content Guidelines for AI/Spec-Driven Approach

### Text Formatting
- Use **bold** for AI tool names and specification terms
- Use *italics* for new AI/Spec methodology terms on first use
- Use `code` formatting for AI tool commands and spec references
- Use ```code blocks``` for multi-line specification examples

### Research-Concurrent Citations
All sources must follow APA style with additional metadata for research-concurrency:

```markdown
## Research Sources
- Primary sources with access dates (critical for AI research)
- AI tool outputs with validation status
- Specification references with version numbers
- Peer review notes and expert validation
```

### AI Quality Markers
- `[AI-Generated]` tag for AI-created content
- `[Human-Verified]` for content verified by human editors
- `[Needs-Review]` for content requiring additional validation
- `[Current-as-of: YYYY-MM-DD]` for time-sensitive information

### Accessibility with AI Integration
- AI-generated alt text must be reviewed for accuracy
- AI-created transcripts need human verification
- Ensure AI-generated content meets accessibility standards

## File Organization for AI/Spec-Driven Workflow

### Naming Convention
`[module-number]-[chapter-number]-[section-number]-[descriptive-title]-[spec-version].md`

Example: `01-01-01-introduction-ai-content-creation-v1.md`

### Related Files
Each section should potentially include:
- `[section-name].md` - Main content
- `[section-name].research.md` - Research notes and sources
- `[section-name].validation.md` - Quality validation records
- `[section-name].spec-ref.md` - Specification alignment documentation

## Quality Standards for AI/Spec-Driven Content

### AI Tool Accuracy
- All AI-generated claims must be independently verified
- Source attribution maintained for AI-assisted content
- Clear distinction between AI and human-generated content

### Specification Compliance
- Content aligns with defined specifications
- Specification references properly linked and up-to-date
- Deviations from spec documented and justified

### Research Currency
- Information reflects current state of knowledge
- Time-sensitive information properly flagged
- Regular updates scheduled for fast-changing domains

## Validation Process

### Automated Checks
- Specification compliance verification
- AI content tagging validation
- Cross-reference accuracy
- Accessibility standard compliance

### Human Review
- Technical accuracy verification
- AI output validation
- Specification alignment confirmation
- Quality and tone assessment

This template provides a framework for creating content that fully integrates AI tools with specification-driven methodologies while maintaining high quality standards and proper research-concurrency practices.
