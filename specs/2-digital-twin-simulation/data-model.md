# Data Model: Digital Twin Simulation (Gazebo & Unity)

## Entity: Simulation Environment
- **Properties**:
  - name: string (environment identifier)
  - physics_properties: object (gravity, friction, etc.)
  - lighting_conditions: object (ambient, directional light)
  - spatial_layout: object (dimensions, boundaries)
- **Relationships**:
  - Contains multiple Robot Models
  - Contains multiple Environment Objects
- **Validation**:
  - Must have valid physics parameters
  - Spatial dimensions must be positive
- **State transitions**: N/A (configuration-based)

## Entity: Robot Model
- **Properties**:
  - name: string (robot identifier)
  - geometry: object (visual and collision meshes)
  - joints: array (joint configurations)
  - sensors: array (attached sensor configurations)
  - dynamic_properties: object (mass, inertia, etc.)
- **Relationships**:
  - Belongs to one Simulation Environment
  - Contains multiple Sensor Data streams
- **Validation**:
  - Must have valid URDF/SDF representation
  - Joint configurations must be physically plausible
- **State transitions**: N/A (configuration-based)

## Entity: Sensor Data
- **Properties**:
  - type: enum (LiDAR, Depth, IMU, etc.)
  - timestamp: datetime (when data was generated)
  - data_payload: object (sensor-specific data format)
  - source_robot: string (robot identifier)
- **Relationships**:
  - Generated by one Robot Model
  - Associated with Simulation Environment
- **Validation**:
  - Data format must match sensor type
  - Timestamp must be within simulation time
- **State transitions**: N/A (time-series data)

## Entity: Human Interface
- **Properties**:
  - interface_type: enum (Unity-based, Web-based, etc.)
  - controls: array (available control options)
  - visualization_elements: array (displayed elements)
  - connection_status: enum (connected, disconnected)
- **Relationships**:
  - Connected to Simulation Environment
  - Interacts with Robot Models
- **Validation**:
  - Must maintain real-time performance
  - User inputs must be validated
- **State transitions**: connected ↔ disconnected

## Entity: Environment Objects
- **Properties**:
  - name: string (object identifier)
  - type: enum (obstacle, goal, interactive, terrain)
  - position: object (x, y, z coordinates)
  - physical_properties: object (mass, friction, etc.)
- **Relationships**:
  - Belongs to one Simulation Environment
  - Interacts with Robot Models
- **Validation**:
  - Position must be within environment bounds
  - Physical properties must be physically plausible
- **State transitions**: static ↔ dynamic (if object becomes movable)

## Validation Rules from Requirements
- All physics parameters must conform to real-world values
- Sensor data must be generated at appropriate frequencies
- Human interface must maintain 30+ FPS performance
- All examples must be reproducible in Claude Code environment

## State Transitions
- Human Interface: connected/disconnected (based on connection status)
- Environment Objects: static/dynamic (if interactivity changes)

## Relationships
- Simulation Environment contains Robot Models and Environment Objects
- Robot Models generate Sensor Data
- Human Interface connects to Simulation Environment
- Human Interface interacts with Robot Models and Environment Objects
